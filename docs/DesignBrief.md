PART 2: COMPLETE REDESIGN DESIGN BRIEF
Project Overview
Client: Boost by FC Rosengård
Project: Full website redesign
Site: boostbyfcr.se
Language: Swedish (primary), English consideration for future internationalisation
Classification: Non-profit / Social enterprise / Workforce development

1. Current State Audit & Pain Points
Visual & Brand Issues

Logo feels dated — heavy collegiate typography with no modern polish
Hero images are over-compressed, low contrast, and suffer from a "cloud wave" SVG divider that reads as visually amateur
Colour palette is inconsistent — navy (#1C3566) and gold (#F5A623) exist but are poorly applied; green submit buttons feel out of brand
Typography is generic system-default; no deliberate type hierarchy
All sub-pages use the same identical hero banner layout with "Bli en del av Boost" — no page differentiation

UX & Navigation Issues

Two dropdown menus ("Om oss" + "Mer…") create cognitive friction — users don't know where to look
"Mer…" label is vague and non-descriptive (hides Press, Jobs, Privacy)
No search functionality
No language toggle (limits accessibility for Malmö's multilingual demographics)
No clear primary CTA in the global navigation
Mobile responsiveness is technically present but visually unpolished
Contact page is the only conversion point — no inline CTAs on track/service pages

Content & Architecture Issues

"Våra spår" is buried 3 clicks deep (Om oss → Våra spår) — this is the core product
FAQ is duplicated on the homepage (the same 6 questions appear 3× in the DOM)
Resurser page is disconnected from the core user journey
No success metrics displayed prominently (200–300 placements/year is compelling — it's hidden)
No blog, news, or impact reporting section


2. Redesign Goals

Establish Boost as a modern, trustworthy, and energetic social brand
Reduce time-to-action for all three primary user groups (participants, employers, students)
Surface the three programme tracks as the primary navigation driver
Increase emotional resonance with Malmö's diverse youth demographic (18–29)
Improve accessibility (WCAG 2.2 AA compliance)
Create a scalable design system for future content growth

6. Page-by-Page Design Specifications
Homepage
The hero transitions from a static photo to a split-screen layout: left side is a large bold headline ("Din väg. Ditt tempo. Din framtid.") with a short value statement and two CTA buttons (primary: "Anmäl dig" in gold, secondary: "Läs om programmen" in outlined navy). Right side features a looping, auto-playing video reel of real participants in 2:3 aspect ratio with a subtle gradient overlay. Below the fold, three card components for the three tracks render side-by-side on desktop (full-bleed stacked on mobile) with bold track icons, one-sentence descriptions, and a "Läs mer" arrow link. An Impact Counter section renders animated number roll-ups (200–300 jobb per år, 100–150 börjar studera, 8/10 nöjdhet) against the dark navy background — this transforms a buried FAQ answer into a bold conversion argument. A testimonials carousel follows, using large-format pull-quote typography with participant photos in rounded frames.
Programme Track Pages (Studiespåret / Arbetsspåret / Hälsospåret)
Each track gets a visually distinct accent colour (Study = Teal, Work = Gold, Health = Emerald). The page structure follows: large hero with track name + tagline → "What you get" in a 2-column feature list → Step-by-step process timeline (how to get started) → Testimonial specific to that track → FAQ accordion → sticky bottom CTA bar on mobile ("Anmäl dig till detta spår →").
Företag (Companies)
Redesigned with a B2B tone shift. Opens with a bold headline "Hitta din nästa stjärna" and a clear value proposition. The 6-step recruitment process infographic is rebuilt as an animated horizontal stepper. Three benefit pillars render as icon + headline + body text in a three-column grid. A "Partners vi jobbar med" logo strip adds social proof. The contact CTA is a prominent form embedded directly on the page (not a redirect), with fields for company name, contact person, phone, and a brief note.
Om Boost (Vem vi är / Vad vi gör)
The "About" cluster is unified under a sticky sub-nav tab bar (common in NGO and social enterprise sites). Content sections use a timeline treatment for the organisation's history since 2003. Staff cards receive a proper grid layout with headshots, names, roles, and optional LinkedIn links. The Agenda 2030 section becomes a visually rich icon-grid layout, each SDG goal shown with the official UN colour and number.
Kontakt
The contact page is split into two columns: a warm introductory message on the left with staff photos and phone/email, and the form on the right. A Google Maps embed replaces the plain text address. The form gains proper floating-label inputs, real-time validation feedback, and a success confirmation animation.

7. Component Library
The design system should define and document the following reusable components: Navigation (desktop mega-menu + mobile drawer), Hero (variant A: split; variant B: full-bleed with overlay), Programme Card (with track colour theming), Impact Counter (animated stat), Testimonial (quote + photo), Timeline Step, Accordion / FAQ, CTA Banner, Form Input (text, textarea, select, checkbox), Button (primary, secondary, ghost, icon-only), Badge / Tag, Alert (info, success, error), Footer (two-column links + partner logos).

8. Accessibility Requirements

WCAG 2.2 AA conformance minimum (AAA for colour contrast on body text)
All images require descriptive alt text (no decorative-image exceptions for hero photos showing people)
Form fields must have visible labels and ARIA descriptors
Skip-to-content link in DOM order
Focus indicators: 2px offset outline using the gold accent
Minimum touch target 44×44px on all interactive elements
Reduced motion media query for all CSS animations
Language attribute lang="sv" on <html>, with regional language toggle for Arabic and Somali (Malmö's top non-Swedish languages) as a phase-2 enhancement


9. Motion & Interaction

Page transitions: fade-in + subtle upward slide (16px, 300ms ease-out) on content sections as they enter viewport
Hero stat counters: count-up animation triggered on scroll-into-view
Programme cards: lift shadow on hover (translateY -4px, 200ms)
CTA button: scale 1.02 on hover + ripple effect
Mobile navigation: slide-in drawer from left (360px max-width) with backdrop blur
No auto-playing videos with sound; all video has visible controls and captions


10. Recommended Tech Stack
Frontend Framework: Next.js 14 (App Router)
This is the definitive choice for this project. Server-side rendering and static generation provide excellent SEO — critical for a non-profit that relies on discoverability. The App Router enables layout segments that allow each track page to share structure while maintaining unique themes. The React ecosystem provides the component reusability needed for the design system.
Styling: Tailwind CSS v3 + shadcn/ui
Tailwind's utility-first approach accelerates delivery and enforces the 4px spacing grid by default. shadcn/ui provides accessible, unstyled component primitives (dialogs, accordions, tabs) that map directly to the component library defined above. Custom design tokens (colours, typography scale) are configured in tailwind.config.ts.
CMS: Sanity v3 (Headless CMS)
The client team needs to update programme descriptions, job postings, testimonials, and impact numbers without developer involvement. Sanity's Studio is highly customisable, its GROQ query language is performant, and it supports structured content modelling that prevents editors from breaking layout. The real-time collaborative editing is a strong operational benefit for a small team.
Animation: Framer Motion
Best-in-class React animation library. Handles the viewport-triggered count-up animations, page transitions, card hovers, and the mobile drawer open/close states. The useInView hook + variants pattern aligns perfectly with the scroll-triggered impact stats.
Forms: React Hook Form + Zod
React Hook Form delivers performant, uncontrolled form state with minimal re-renders. Zod provides TypeScript-first schema validation that runs both on client (for real-time feedback) and server (in Next.js Server Actions), eliminating the need for a separate form API endpoint.
Maps: Mapbox GL JS (or react-leaflet for budget constraint)
For the Contact page map embed. Mapbox provides a better-styled, more customisable experience than Google Maps and offers a generous free tier.
Email / Contact Form Backend: Resend + React Email
Replaces the current generic form processor. Resend is a modern transactional email API with a React Email template system, allowing the team to design and maintain beautiful, branded confirmation emails.
Hosting & Infrastructure: Vercel
Natural pairing with Next.js. Provides edge CDN, automatic preview deployments per PR (invaluable for client review), built-in analytics, and zero-config CI/CD from GitHub.
Analytics: Vercel Analytics + PostHog
Vercel Analytics for performance and Core Web Vitals. PostHog for product analytics (session replays, funnel tracking from homepage → programme page → sign-up CTA) without requiring cookie consent under Swedish/EU law for privacy-preserving mode.
Version Control & CI/CD: GitHub + GitHub Actions
Linting (ESLint + Prettier), type checking (tsc --noEmit), and Lighthouse CI score gating on every pull request.
Package Manager: pnpm
Faster installs, efficient disk usage via the content-addressable store — important for CI/CD speed.
